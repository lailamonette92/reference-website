Description list = <dl> ,<dt>, <dd>

A description list is a list of terms, with a description of each term.

The <dl> tag defines the description list, the <dt> tag defines the term (name), and the <dd> tag describes each term:

Coffee
- black hot drink
Milk

a tag

The <a> tag defines a hyperlink, which is used to link from one page to another. The most important attribute of the <a> element is the href attribute, which indicates the link's destination.

By default, links will appear as follows in all browsers:

  • An unvisited link is underlined and blue
  • A visited link is underlined and purple
  • An active link is underlined and red

img tag

The <img> tag is used to embed an image in an HTML page.

Images are not technically inserted into a web page; images are linked to web pages. The tag creates a holding space for the referenced image.

The <img> tag has two required attributes:

  • src - Specifies the path to the image
  • alt - Specifies an alternate text for the image, if the image for some reason cannot be displayed

Figure tag

The <figure> tag specifies self-contained content, like illustrations, diagrams, photos, code listings, etc.

While the content of the <figure> element is related to the main flow, its position is independent of the main flow, and if removed it should not affect the flow of the document.

Tip: The <figcaption> element is used to add a caption for the <figure> element.

Header = <header>

The header is the introductory content usually at the top of the page and would include a logo or company name and main navigation. Headers are also within sections and articles to introduce the content for headings, subheadings, author, etc..

footer = <footer>

Many webpages end their pages with a footer at the bottom. This often includes lists of links, copyright information, extra information about the company, terms and services.

Main = <main>

The main element wraps around the main content of the page. It’s only used once per page and it tells the browser that this is where the main content is.

section = <section>

Section is used to wrap around related pieces of content. Usually each section has its own heading.

article = <article>

The article element represents a self- contained composition in a web page, which is intended to be independently distributable or reusable. Examples include blog post, product card, interactive widget, etc...

navigation = <nav>

The nav element represents a section of a page whose purpose is to provide navigation links, either within the current document or to other documents.

Landmark Roles

ARIA has a set of landmark roles to help those using screen readers to jump directly to specific sections within our sites.

<header role = "banner" >
- This role is for defining an element as the masthead, added to the primary
<nav role ="navigation">
- This role is for defining an element as the primary navigation, added the the top <nav>.
<main role ="main">
- This role is for defining an element as being the main content of the page, added to the <main>
<aside role ="complimentary">
- This role is for defining an element as being content related to the primary content, usually added to an <aside>
<footer role ="contentinfo">
- This role is for defining an element as information about the content: e.g. copyright, terms, privacy policy, usually added to a <footer>

Aria-Label

Web Accessibility Initiative (WAI) has a specification called Accesible Rich Internet Apllication (ARIA)

Adding aria-label attribute provides the user with a non-visual label that will only be announced by screen readers.

image description

Always describe your images with the alt attribute. If purely decorative you can leave it empty and apply role=“presentation”

The alt attribute has two purposes:

  • To describe the photo if it doesn’t download
  • To describe the photo to someone who cannot see the image, maybe someone using a screen reader.

Font Properties

There are a number of font properties that has to do with the thickness of the text, sizing, and what the actual font family is. Below are a few of the most common font properties.

font-family

Specifies which font family to use for whatever text you're selecting in your CSS

font-size

Changes the size of the font.

font-weight

Changes the thickness of the font.

font-style

Change the way the font is displayed e.g. italicize, oblique, or normal.

line-height

Changes the height of a line

Font Shorthand

This isn't a property that you can edit, but this is a way for you to apply all the above properties into one. The way you would do this is: font: font-style font-variant font-weight font-size/line-height font-family;

font-size and font-family is required for this shorthand, but if any of the other values are missing, their default value is used.

Text Decorations

Text can be decorated / styled in a lot of different ways. Here's a few of the most common ways to style text.

text-transform

Allow you to change the case of your text such as uppercase, lowercase, or capitalize. (text-transform: uppercase;)

text-decoration

Add text-decoration to your text, such as underlines, overlines, dashed lines etc. You can also remove text-decorations from your links to make them look cleaner. (text-decoration: overline wavy;)

text-align

Specifies the alignment of your text (center, left, right) (text-align: right;)

text-indent

Specifies how much indentation to apply to your text. (text-indent: 50%;)

text-shadow

Adds a shadow effect to your text. (text-shadow: 2px 2px;)

Link States (default, visited, hover, focus)

Link states allows you to go in depth with user interaction of links. You can style exactly how a link looks depending on a number of factors:

  • default: The way the link looks by default, before a user has clicked on a link.
  • visited: How a link looks if the user has visited it before. Much like how on Google, if you've visited a link before, it shows up as purple instead of blue
  • hover: How the link looks when a cursor is hovering over it.
  • focus: How the link looks when the keyboard is focused on the link.

You can use these link states in your CSS by selecting a then attaching a colon onto it, and select the state you'd like to change, like: a:hover

Margin

Margin allows you to generate space around an element's content, outside of the defined borders.

There are multiple ways to define the margins of an element. The shorthand way uses margin: top right bottom left;. If the margin property has one value (margin: 25px;), all four margins will be 25px. If the margin property has two values (margin: 25px 50px;), then the top and bottom margins will be 25px, and the right and left margins will be 50px. You can also explicitly define each of these properties, like so:

  • margin-top: 8px
  • margin-right: 12px
  • margin-left: 2em
  • margin-top: 0%
Margin uses length properties (px, cm, pt, etc..), inherit, %, and auto to define spacing.
